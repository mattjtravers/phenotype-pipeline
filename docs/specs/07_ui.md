# UI Specs

## Input

- [x] **UI-UI-001**: The system shall present a file upload widget accepting `.vcf` files.
- [x] **UI-UI-003**: If the user-uploaded file has an extension other than `.vcf`, then the system shall display an inline error message and shall not submit the file for prediction.
- [x] **UI-UI-004**: If the user-uploaded file exceeds 50 MB, then the system shall display an inline error message and shall not submit the file for prediction.
- [x] **UI-UI-005**: If the user-uploaded VCF contains more than one sample, then the system shall display an inline error message and shall not submit the file for prediction.

## Prediction Request

- [x] **UI-UI-008**: When the user submits a valid VCF input (uploaded file or selected sample), the system shall dispatch the prediction request as an HTTP POST containing only the VCF content to the Lambda API Gateway endpoint configured via `PHENO_API_ENDPOINT`.
- [x] **UI-UI-009**: While a prediction request is in progress, the system shall display a loading indicator and disable the submit button.
- [x] **UI-UI-010**: If the prediction request fails or times out, then the system shall display an error message describing the failure and re-enable the submit button.
- [x] **UI-UI-015**: When the prediction request returns an error response in the format defined by `DEPLOY-BE-022` (`{"error": <code>, "detail": <message>}`), the UI shall map the `error` code to a human-readable message: `MODEL_UNAVAILABLE` → "The prediction service is starting up or unavailable — please try again in a moment."; `INVALID_VCF` → "The VCF file could not be processed (multi-sample file, malformed format, or no usable variants)."; `INVALID_INPUT` → "The request was rejected as malformed — please reload and try again."; `INFERENCE_FAILED` / `INTERNAL_ERROR` → "The prediction service encountered an error — please try again or contact support." Unknown codes shall fall through to the generic `UI-UI-010` message.

## Sample File Input

- [x] **UI-UI-016**: When the `samples/` directory exists relative to the app script at startup, the system shall display a "Don't have a file? Try a sample" expander in the input form, expanded open by default.
- [x] **UI-UI-017**: When the `samples/` directory is absent at startup, the system shall hide the "Try a sample" expander and display no error.
- [x] **UI-UI-018**: The "Try a sample" expander shall present a radio group whose first option is "None — use uploaded file" followed by one labelled option per file in `samples/` whose name matches the glob `sample_snp_[0-9]*.vcf`. The label for each such file shall be "Sample N" where N is the numeric suffix (e.g., `sample_snp_1.vcf` → `"Sample 1"`). Other `.vcf` files present in `samples/` (e.g., upload-demo files) are not shown.
- [x] **UI-UI-019**: On page load, the sample radio group shall have "None — use uploaded file" selected by default (no sample pre-selected).
- [x] **UI-UI-020**: When the user selects a sample (any option other than "None") and no file has been uploaded, the system shall use that sample file as the prediction input.
- [x] **UI-UI-021**: When the user has both uploaded a file and selected a sample (any option other than "None"), the system shall use the uploaded file as the prediction input and disregard the sample selection.
- [x] **UI-UI-022**: When a sample file is used as the prediction input, the system shall bypass client-side validation (extension, size, and sample-count checks).

## Results Display

- [x] **UI-UI-011**: When a prediction result is returned, the system shall display the predicted ancestral population label, the confidence score as a percentage with a progress bar, and the top contributing markers.
- [x] **UI-UI-012**: The system shall display the top contributing markers (as defined in PRED-DATA-002) in a ranked table showing variant ID, chromosome, position, and SHAP contribution value.
- [x] **UI-UI-014**: The system shall display the top contributing markers as a horizontal bar chart of SHAP contribution values, sorted by absolute contribution descending.
- [x] **UI-UI-013**: The system shall provide a download button that exports the full `PredictionResult` (as defined in PRED-PROC-009) as a JSON file.
