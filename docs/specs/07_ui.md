# UI Specs

## Input

- [ ] **UI-UI-001**: The system shall present a file upload widget accepting `.vcf` files.
- [ ] **UI-UI-002**: The system shall present a dropdown for the user to select the target phenotype before submitting. The dropdown options shall be fetched at startup from the Lambda `/labels` endpoint (as defined in DEPLOY-BE-013), which returns the deployed model's `label_encoder.json` values.
- [ ] **UI-UI-006**: If the `/labels` endpoint fetch fails or times out at startup, then the system shall disable the phenotype dropdown and the submit button, and shall display an inline error message identifying the failure.
- [ ] **UI-UI-003**: If the uploaded file has an extension other than `.vcf`, then the system shall display an inline error message and shall not submit the file for prediction.
- [ ] **UI-UI-004**: If the uploaded file exceeds 50 MB, then the system shall display an inline error message and shall not submit the file for prediction.
- [ ] **UI-UI-005**: If the uploaded VCF contains more than one sample, then the system shall display an inline error message and shall not submit the file for prediction.

## Prediction Request

- [ ] **UI-UI-008**: When the user submits a valid VCF file and phenotype selection, the system shall dispatch the prediction request as an HTTP POST to the Lambda API Gateway endpoint configured via `PHENO_API_ENDPOINT`.
- [ ] **UI-UI-009**: While a prediction request is in progress, the system shall display a loading indicator and disable the submit button.
- [ ] **UI-UI-010**: If the prediction request fails or times out, then the system shall display an error message describing the failure and re-enable the submit button.

## Results Display

- [ ] **UI-UI-011**: When a prediction result is returned, the system shall display the predicted phenotype label, the confidence score as a percentage with a progress bar, and the top contributing markers.
- [ ] **UI-UI-012**: The system shall display the top contributing markers (as defined in PRED-DATA-002) in a ranked table showing variant ID, chromosome, position, and SHAP contribution value.
- [ ] **UI-UI-014**: The system shall display the top contributing markers as a horizontal bar chart of SHAP contribution values, sorted by absolute contribution descending.
- [ ] **UI-UI-013**: The system shall provide a download button that exports the full `PredictionResult` (as defined in PRED-PROC-009) as a JSON file.
